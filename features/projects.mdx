# Projects

Projects in Hive help you organize tasks and collaborate with your team. Each project acts as a container for related tasks and allows you to track progress toward specific goals.

## Creating a New Project

To create a project:

1. Navigate to the **Projects** screen from the sidebar
2. Click the **Nuevo Proyecto** button in the top-right corner
3. Fill in the project details:
   - **Project Name** (required)
   - **Description** (optional)
   - **Start Date** (required)
   - **End Date** (required)
   - **Status**: Active, Inactive, or Completed
   - **Project Color**: Choose a color to identify your project visually
   - **Task Area Opacity**: Adjust transparency of the task display area

<Tip>
Choose distinct colors for each project to make them easily identifiable in the projects view and task boards.
</Tip>

## Managing Project Tasks

Each project displays its tasks in a dedicated column view:

- **Expand/Collapse**: Click the project header to show or hide tasks
- **Task Counter**: See the number of completed vs. total tasks at a glance
- **Drag & Drop**: Move tasks between projects by dragging task cards
- **Progress Tracking**: Visual progress bars show completion percentage

### Task Organization

Tasks within a project are displayed with:
- Task title and description
- Assigned team members (with avatars)
- Priority badges (Low, Medium, High, Urgent)
- Due dates and remaining time
- Status indicators (Not Started, In Progress, Completed)
- Progress percentage with visual bar
- Associated tags/labels

<Tip>
Use the search bar to filter projects by name. The clear button (X) appears automatically when you start typing.
</Tip>

## Editing Projects

To modify a project:

1. Click the **Edit** icon on the project header
2. Update any field (name, dates, status, color, description)
3. View the project creator information (read-only)
4. Click **Actualizar Proyecto** to save changes

<Warning>
Changing project dates doesn't automatically adjust task due dates. You may need to update individual tasks separately.
</Warning>

## Project Settings

### Color Customization

Projects support full color customization:
- Choose from the color picker for any shade
- The selected color appears on project headers and task cards
- Opacity settings control the transparency of the task display area

### Team Assignment

Team members are assigned at the **task level** rather than the project level. This allows for:
- Flexible resource allocation across tasks
- Multiple team members on different tasks within the same project
- Clear accountability for specific deliverables

## Deleting Projects

<Warning>
Deleting a project is permanent and will affect all associated tasks.
</Warning>

To delete a project:

1. Open the project edit modal
2. Click the **Eliminar Proyecto** button (red)
3. Confirm the deletion in the confirmation dialog

## Best Practices

<AccordionGroup>
  <Accordion title="How should I organize projects?">
    Organize projects by major initiatives, clients, or departments. Keep projects focused on specific outcomes rather than creating overly broad project categories.
  </Accordion>

  <Accordion title="When should I mark a project as Inactive vs. Completed?">
    Use **Inactive** for projects that are temporarily paused but may resume later. Use **Completed** for projects that have reached their end goals and won't be reopened.
  </Accordion>

  <Accordion title="Can I move tasks between projects?">
    Yes! Use the drag-and-drop feature in the Projects view to move tasks between project columns. You can also change the project assignment in the task edit modal.
  </Accordion>

  <Accordion title="How many projects should I create?">
    Create as many projects as needed to keep work organized. However, having too many active projects can make it harder to focus. Consider marking older projects as Completed to keep your workspace clean.
  </Accordion>
</AccordionGroup>
